# COYL Desktop — macOS Menu Bar EAP Coordinator

A small, always-running menu bar app that makes a Mac into an EAP
edge device. Registers with `coyl.ai`, listens for actions from
connected LLM partners, executes them via AppleScript, Shortcuts,
notifications, voice TTS, and friends.

This directory holds the Swift source. The Xcode project itself is
generated by the founder once (steps below); the eleven files in
`COYLDesktop/` become the project's targets.

---

## Architecture (eleven files)

```
COYLDesktop/
  COYLDesktopApp.swift        @main + NSApplicationDelegate
  MenuBarController.swift     NSStatusBar item + dropdown menu
  EAPCoordinator.swift        Talks to coyl.ai/api/eap/v1/*
  ActuatorRouter.swift        actuator → AppleScript / Shortcut / etc.
  SensorPublisher.swift       Periodic sensor publish loop
  DeviceRegistration.swift    Builds + posts /device/register
  Auth.swift                  Keychain storage + sign-in URL handoff
  ConsentWindow.swift         SwiftUI scope-grant UI
  Models/
    EAPAction.swift           Wire shape of an EAP action + outcome
    EAPDevice.swift           Device registration + manifest types
    EAPScope.swift            Scope vocabulary + grant cache
  Info.plist                  LSUIElement + URL scheme + ATS pinning
  COYLDesktop.entitlements    No-sandbox + Apple Events + Network
```

Launch ordering (see `COYLDesktopApp.swift`):

1. `applicationWillFinishLaunching` — register `coyl-desktop://` URL handler
2. `applicationDidFinishLaunching` — install menu bar, kick coordinator + sensor loop, request notification permission
3. `EAPCoordinator.start()` — async POST `/api/eap/v1/device/register`, then on success kick the 30s poll timer against `/api/eap/v1/devices/<id>/pending-actions`

## Founder Xcode steps (one-time)

1. **Create the project**

   - Open Xcode → **File → New → Project → macOS → App**
   - Product name: `COYL Desktop`
   - Team: your Developer ID team
   - Organization identifier: `ai.coyl`
   - Bundle Identifier: `ai.coyl.desktop`
   - Interface: **SwiftUI**, Language: **Swift**
   - Storage: **None** (we use Keychain + UserDefaults directly)
   - Tests: **off** (add later if needed)

2. **Replace the auto-generated files**

   Delete Xcode's default `ContentView.swift` and `*App.swift`. Drag the contents of `apps/desktop-macos/COYLDesktop/` into the project navigator. Make sure "Copy items if needed" is OFF — these files live in the repo, not the Xcode bundle.

3. **Project settings (target → General)**

   - Minimum Deployments: **macOS 13.0**
   - Identity → Display Name: `COYL`
   - Identity → Bundle Identifier: `ai.coyl.desktop`
   - App Icons and Launch Screen → App Icon: drop in once the founder has assets

4. **Project settings (target → Signing & Capabilities)**

   - Signing → Automatically manage signing: ON, team = founder's Apple Developer team
   - Click **+ Capability** and add the following:
     - **App Group**: `group.com.coyl.shared`
     - **Outgoing Connections (Client)** (already on by default)
     - **User Notifications**
   - Replace the auto-generated `.entitlements` file with `COYLDesktop.entitlements` from this directory.
   - In **Info → Custom macOS Application Target Properties** (or by setting the `INFOPLIST_FILE` build setting), point to `COYLDesktop/Info.plist`. Don't let Xcode autogenerate one.

5. **Info.plist privacy strings**

   These are already present in the checked-in `Info.plist`. Verify Xcode picked them up:

   - `LSUIElement = YES` — removes the Dock icon
   - `NSAppleEventsUsageDescription`
   - `NSCalendarsUsageDescription` / `NSCalendarsFullAccessUsageDescription`
   - `NSUserNotificationsUsageDescription`
   - `CFBundleURLTypes` with scheme `coyl-desktop`

6. **App Transport Security pinning — REPLACE BEFORE FIRST BUILD**

   `Info.plist` ships with placeholder SPKI hashes:

   ```
   REPLACE_WITH_COYL_AI_LEAF_SPKI_SHA256_BASE64
   REPLACE_WITH_COYL_AI_BACKUP_LEAF_SPKI_SHA256_BASE64
   ```

   Get the real values by running, on a Mac with `openssl`:

   ```bash
   openssl s_client -servername coyl.ai -connect coyl.ai:443 < /dev/null 2>/dev/null \
     | openssl x509 -pubkey -noout \
     | openssl pkey -pubin -outform der \
     | openssl dgst -sha256 -binary \
     | base64
   ```

   Paste the result into the first `<string>` under `NSPinnedLeafIdentities`. Repeat with the backup cert (rotate every quarter; keep both pins live during the cutover window).

7. **Build + run locally**

   - Hit **⌘R**. The Dock won't show an icon — look for the menu bar.
   - On first launch macOS asks for notification permission; tap Allow.
   - Use **Sign in…** from the menu to test the browser handoff. (The web side needs to redirect to `coyl-desktop://auth?token=...&userId=...` on success.)

## Actuator coverage (≈80% of EAP spec)

Implemented in `ActuatorRouter.swift`:

| EAP actuator           | macOS implementation                | Notes |
|---|---|---|
| `notification`         | `UNUserNotificationCenter`          | Banner + sound. |
| `voice_tts`            | `NSSpeechSynthesizer`               | Works without AVAudioSession boilerplate. |
| `open_app`             | `NSWorkspace.openApplication`       | Param: `bundleId`. |
| `open_url`             | `NSWorkspace.open(URL)`             | Param: `url`. |
| `run_shortcut`         | `/usr/bin/shortcuts run <name>`     | User must have created the Shortcut. |
| `applescript_execute`  | `NSAppleScript`                     | Each target app's first use triggers a TCC prompt. |
| `dim_screen`           | AppleScript brightness-down keystrokes | Coarse (~6% per press); precise control needs private CoreDisplay APIs we don't link. |
| `do_not_disturb_toggle`| Bridged to a user-installed Shortcut | macOS 14 removed the public toggle path. |

**Documented gaps** (return `outcome=failed:unsupported`):

- `send_message` / `send_message_irreversible` — macOS doesn't expose programmatic iMessage send. The LLM should re-emit as `run_shortcut` against a user-installed "Send iMessage" Shortcut.
- `modify_system_preferences` — blocked since macOS 14. Surface as a notification asking the user to make the change manually.
- Reading another app's UI state without an explicit Accessibility grant.

## Sensor coverage

Implemented in `SensorPublisher.swift`. Snapshot published to `POST /api/eap/v1/sensor/<deviceId>/publish` every 60s (tunable via UserDefaults `COYLDesktop.publishIntervalSeconds`).

| EAP sensor                  | macOS source | Permission gate |
|---|---|---|
| `screen_state`              | `CGSessionCopyCurrentDictionary` | None |
| `battery`                   | `IOPSCopyPowerSourcesInfo` | None |
| `foreground_app`            | `NSWorkspace.frontmostApplication` | None |
| `calendar_meeting_density`  | `EventKit` event count for today | Calendar |
| `typing_pace`               | `NSEvent.addGlobalMonitorForEvents` (keyDown) over 60s window | Accessibility |

## Server contract dependencies (TODO before ship)

The macOS coordinator polls + publishes against three endpoints. Two already exist on `coyl.ai`; one needs to be shipped server-side:

- ✅ `POST /api/eap/v1/device/register` (live)
- ✅ `POST /api/eap/v1/action/outcome` (live)
- ⚠️ `GET  /api/eap/v1/devices/<deviceId>/pending-actions` (NOT YET) — required for the poll loop
- ⚠️ `POST /api/eap/v1/sensor/<deviceId>/publish` (NOT YET) — required for the sensor stream

The polling endpoint should return:

```json
{
  "actions": [
    {
      "id": "...",
      "executionToken": "...",
      "actuator": "notification",
      "params": { "title": "...", "body": "..." },
      "scopeRequested": "edge:laptop:notification",
      "reasoning": "...",
      "confidence": 0.83,
      "willExecuteAt": "2026-05-22T08:00:00Z",
      "ttlSeconds": 60,
      "llmPartnerId": "anthropic-claude-sonnet-3-7"
    }
  ]
}
```

Both endpoints must accept the same Bearer token shape as the rest of EAP: a Clerk session JWT in the bootstrapping window, swapped for an EAP device token issued at first `/device/register` time.

## Distribution

Two paths. The repo defaults to **path A** because path B would force app-sandboxing and break several actuators.

### Path A: Developer ID + notarytool + direct download (preferred)

```bash
# Archive from Xcode (Product → Archive), then:
xcrun notarytool submit COYLDesktop.zip \
  --apple-id "founder@coyl.ai" \
  --team-id  "<TEAM_ID>" \
  --password "@keychain:AC_PASSWORD" \
  --wait

xcrun stapler staple COYLDesktop.app
ditto -c -k --keepParent COYLDesktop.app COYLDesktop-0.1.0.zip
```

Upload `COYLDesktop-0.1.0.zip` to `coyl.ai/download/macos`.

### Path B: Mac App Store

Requires `com.apple.security.app-sandbox = true`, which:

- Disables AppleScript control of other apps (would gate every actuator)
- Disables forking `/usr/bin/shortcuts` (breaks `run_shortcut`)
- Restricts global NSEvent monitors (breaks `typing_pace`)

If you need MAS distribution, ship a sandbox-compatible variant with reduced actuator coverage (~30%) under a different bundle ID (e.g. `ai.coyl.desktop.mas`).

## Local development

Set `COYL_BASE_URL=http://localhost:3000` in the Xcode scheme's environment to point the coordinator at a local Next.js dev server. The `Info.plist` ATS exception domain already permits `localhost`.

For quick wire smoke-testing without Xcode:

```bash
cd apps/desktop-macos/COYLDesktop
swiftc -target arm64-apple-macosx13 -parse \
  COYLDesktopApp.swift MenuBarController.swift EAPCoordinator.swift \
  ActuatorRouter.swift SensorPublisher.swift DeviceRegistration.swift \
  Auth.swift ConsentWindow.swift \
  Models/EAPAction.swift Models/EAPDevice.swift Models/EAPScope.swift
```

The `-parse` flag stops at type-checking — no binary is produced, which is the right depth for catching breakage in the model layer without dragging in the whole AppKit/EventKit link graph.
